Heart Attack Prediction

Data Source: Kaggle.com | Library: Pandas, Matplotlib, Seaborn

Data Explorartion / EDA

I have used the data from website Kaggle to predict the Heart Attack rate of the patient based on their biological dataset such as their age, gender, cholesterol amount etc. Below is the head of the data that I used.

After checking up with the data, see if there is any null values and see how many unique values for each columns, I have changed the categorical values into numeric values using mapping function to visualize those into plots. Below image shows the distribution of each categorical features.

From the graphs above, I was able to find out that all the features contribute to heart attack at least in certain level

Among all the categorical features, I have decided to get statistics of "gender" feature it is easy to comprehend for people who sees this project and this is the feature which people mostly want from the data.

From the statistic of "gender" feature above, it was clear to see that male has higher possibility of having a heart attack than that of female.

After the visualization of the categorical features, I have decided to look into the numeric categorical features such as age, chol, and etc. Below graphs represent the distribution of each numeric features.

KDE Plot represents the data using a continuous probability density curve. The more two kde plots overlap, the less difference between those two. As I mentioned it before, from the KDE plots I was able to see that every features have certain effect on heart attack rate. Among all the features, I was able to find out that "cholesterol rate" feature has the least amount of impact on heart attack.

If there is a significant difference between two features, it is better to illustrate the difference with the graphs. However, if there two features on the graph is simillar, I thought it would be better to show the heatmap to illustrate the detailed difference between them. Below shows the heatmap(Correlation) of the continuous features.

From the Heatmap I was able to figure out that "age" and "thalachh" features have the highest correlation and "trtbps" and "thalachh" have the least. The annotation of the Heatmap shows that "age" and "thalachh" features are correlated with the value of -0.4, which means that whenever age increases, thalachh decreases and vice versa.

Among all the continuous features, I have decided to pick one feature which was "age" to plot a histogram to see the distribution. Below illustrates the histogram and statistic of "age" feature.

I have excluded the 20s' heart attack rate because there was not enough data. The amount of data is unbalanced between the (30s-60s) and (30s and 70s). Just by looking at the numbers of "more chance of heart attack", 50s have the highest, but they have the highest number of "less chance of heart attack" at the same time. With the statistic, I was able to conclude that the most vulnerable age range was surprisingly 40s.

Prediction

Above picture is a scatter plot of the two most important features (Highly contributes to predict the output) of this data which are "thalachh", and "oldpeak". As you can see that data is hard to be separated linearly. Therefore, I assume the models that use linear method won't perform well.

Above Dataframe illustrates the each model's accuracy scores and like I have assumed before, the methods that use linear method did not perform well, but KNN model performed the best instead.

ROC Curve of the Best Model (KNN)

The ROC Curve shows the trade-off between sensitivity and specificity. If the model gives the curves closer to the top-left corner, it indicates a better performance. On the other hand, if the model gives the curves closer to the 45 degree diagonal line, it represents the less accuracy.

KNN Visualization

In order to visualize the KNN classification, I had to choose two features that are correlated the least and which both correlates the most to the output which are "cp", "oldpeak". (Referred to the heatmap below.)As a result, KNN Classification with only two features gave us 80% of accuracy which is pretty good.
